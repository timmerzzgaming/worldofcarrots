Create or update README.md for the current project.

Include:

- Project purpose (one paragraph)
- Installation instructions (copy-paste ready)
- Usage instructions (most common use case first)
- File structure overview

Rules:

- Inspect the repo first to gather accurate information
- Use concrete examples, not abstract descriptions
- Keep it concise — a new developer should be productive in 5 minutes
- If a README already exists, update it rather than replacing it entirely
