Analyze the repository thoroughly.

Steps:

1. Show directory structure (2-3 levels deep)
2. Identify key files and modules
3. Identify entry points
4. Detect dependencies (package.json, requirements.txt, etc.)
5. Summarize project purpose
6. Identify potential issues (misconfigurations, missing files, dead code)
7. Suggest improvements

Output must be concise and structured.

Usage examples:
- `/analyze` — full repo analysis
- `/analyze what does this project do` — focused summary
- `/analyze suggest improvements` — focus on recommendations
